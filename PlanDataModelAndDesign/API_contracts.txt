Operação: Cadastrar Provedores de Autenticação
Endpoint: POST /api/auth-providers

Descrição: Registra um novo provedor de autenticação no sistema.

Payload (JSON):

JSON
{
  "name": "Google"
}
Resposta de Sucesso (201 Created)
JSON
{
  "id": 1,
  "name": "Google",
  "created_at": "2026-03-14T10:19:49Z"
}

1. API Contract Interfaces (Frontend and Backend)
You can maintain these types in a shared types/auth.ts file or separately in each layer:

TypeScript
// The structure of an individual provider
export interface AuthProvider {
  id: number;
  name: string;
}

// The contract for the bulk creation request

  providers: {
    name: string;
  }[];
}

// The contract for the API response
export interface CreateProvidersResponse {
  message: string;
  count: number;
  providers: AuthProvider[];
}

3. Frontend Call (Next.js - Client Side)
In your component or Server Action, you will consume the API while maintaining type safety:

TypeScript
async function registerProviders() {
  const payload: CreateProvidersRequest = {
    providers: [{ name: 'Google' }, { name: 'Email/Password' }]
  };

  const response = await fetch('/api/auth-providers/bulk', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(payload)
  });

  const data: CreateProvidersResponse = await response.json();
  console.log(`Success: ${data.message} - Total: ${data.count}`);
}
